# Fry OpenClaw Extension

OpenClaw extension for the Fry build orchestration system. Lets you start, monitor, and interact with Fry builds through any OpenClaw messaging channel (Telegram, WhatsApp, iMessage, Discord, etc.).

## Installation

Add to your OpenClaw config (`~/.openclaw/config.yaml` or equivalent):

```yaml
plugins:
  entries:
    fry:
      enabled: true
      source: "/path/to/fry/openclaw-extension"
      config:
        fry_binary: "fry"         # path to fry binary (default: from PATH)
        notifications: "milestones" # all | milestones | errors
```

Enable the tools on your agent:

```yaml
agents:
  - id: main
    tools:
      allow:
        - fry_build_start
        - fry_build_status
        - fry_build_restart
        - fry_build_logs
        - fry_read_progress
        - fry_consciousness_stats
```

## Tools

| Tool | Description |
|------|-------------|
| `fry_build_start` | Start a new Fry build on a project directory |
| `fry_build_status` | Get structured JSON build state (calls `fry status --json`) |
| `fry_build_restart` | Restart a failed/stopped build (--continue, --resume, --simple-continue) |
| `fry_build_logs` | Read recent sprint, heal, or audit logs |
| `fry_read_progress` | Read sprint-progress.txt or epic-progress.txt |
| `fry_consciousness_stats` | Query consciousness pipeline status (memory counts, reflection, identity) |

## System Prompt

The extension loads Fry's system prompt via `fry agent prompt`. This prompt is generated by the Fry binary and includes:

- Fry's identity (from identity.json)
- Complete artifact schema (all .fry/ files, formats, lifecycle)
- Build lifecycle documentation
- Event type reference
- Effort level guide
- Conversation patterns for natural-language interaction

The prompt lives in Go (`internal/agent/prompt.go`) so it stays in sync with Fry's artifact formats.

## Build Watcher

The extension runs a background service that monitors active builds via `fry events --follow --json`. When milestone events occur (sprint complete, build end, errors), notifications are sent through OpenClaw's channels.

## Requirements

- Fry binary installed and accessible from PATH (or configured via `fry_binary`)
- OpenClaw gateway running with at least one messaging channel configured
